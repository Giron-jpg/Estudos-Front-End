Criando formulários

Para criar um formulário usamos a tag form e dentro dela inserimos os elementos necessários. Para criar a caixa de digitação usamos a tag input que é acompanhada de mais uma informação que varia de acordo com o tipo de dado. É importante que todo input seja acompanhado de um label para que haja direcionamento entre o nome do elemento e o input. Para regular melhor alguns formulários é interessante estudar mais RegEx

Para agrupar todo o formulário em um campo usamos a tag fieldset e ainda podemos usar a tag legend para dar nome ao conjunto de dados

Preferências

Alguns parâmetros:

action - cria um vínculo com um arquivo de outra linguagem (posteriormente associado a um script).

autocomplete (on/off) - mostra opções para completar uma caixa em um cadastro. Para melhorar o desempenho deve-se inserir um autocomplete em cada input para identificar qual é o tipo de sugestão que deve aparecer

method - define o método de compartilhamento de envio de dados. Por padrão ele é definido como GET (limite máximo de 3000 letras), com isso os dados preenchidos no formulário são expostos na url (ex.: buscas no google - ajuda no compartilhamento). O método POST (Prefirido no envio de imagens) mascara as respostas do formulário na url porém ainda ficam acessíveis no dev tools > Network > nome.arquivo > headers. Para mais segurança devemos aprender sobre HTTPS

required - normalmente inserido no input, torna obrigatorio o preenchimento de um campo. Esse parametro pode ter especificações como tamanho mínimo e máx (minlength e maxlength)

size - especifica o tamanho da caixa (em caractéres) que recebera o dado

placeholder - preenche previamente o conteudo da caixa com alguma dica se necessário.

checked - em checkbox inicia com a caixa marcada

pattern - torna obrigatório um padrão para envio de um determinado dado. Ex.: pattern="^\(\d{2}\) \d{4,5}-\d{4}$"

Tipos de Input

text - permite inserção de textos

submit - cria um botão que manda os dados

password - omite o texto digitado em "*"

reset - cria um botão que limpa os dados do formulário

date - caixa para datas com calendário

month - cria caixa para mês e ano

number - para inserirvalores numéricos (pode ser usado com os parâmetros max e min para determinar máximos mínimos e step para valores fracionados)

time - para cadastro de horas

tel - cadastro de telefones (não vem com padrão)

email - cadastro de email (também não vem com padrão)

radio - cria uma caixa de marcação permanente (para trocar deve-se criar uma opção alternativa com o mesmo nome)

checkbox - cria uma caixa de marcação
